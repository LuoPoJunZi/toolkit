# 运行级自检清单（v0.1.x）

本文用于在真实 VPS 环境中逐菜单验证功能可用性，避免“看起来有菜单、实际空执行”。

## 1. 自检前准备

- [ ] 使用 `root` 用户登录
- [ ] 更新到最新脚本：`bash <(curl -fsSL https://z.evzzz.com)`
- [ ] 启动工具箱：`z`
- [ ] 记录日志路径：
  - 操作日志：`/opt/luopo-toolkit/logs/action.log`
  - 错误日志：`/opt/luopo-toolkit/logs/error.log`

## 2. 通用通过标准

- [ ] 每个功能必须有明确执行结果（成功或失败提示）
- [ ] 不允许只出现“按回车继续...”且没有过程输出
- [ ] 失败时需有可理解的错误提示（依赖缺失/下载失败/执行失败）
- [ ] 高风险操作需有确认提示（如重启、卸载、批量操作）

## 3. 主菜单逐项检查

### 1. 系统信息查询

- [ ] 输出系统/资源/网络分区信息
- [ ] 无 `numfmt` 报错
- [ ] 核心字段不为空（主机名、内核、内存、磁盘、IPv4）

### 2. 系统全面更新

- [ ] 有更新过程输出
- [ ] 有明确完成状态（成功/失败）

### 3. 系统垃圾清理

- [ ] 有清理过程输出
- [ ] 有明确完成状态（成功/失败）

### 4. 一键脚本中心

- [ ] 脚本列表完整显示
- [ ] 任意脚本可真正执行，不是秒退
- [ ] 下载失败时有明确错误提示

### 5. Docker 管理

- [ ] `1` 安装 Docker 可执行
- [ ] `2/3/4/5` 启停重启状态可执行
- [ ] `6~12` 有有效输出或明确失败提示

### 6. 网络加速管理

- [ ] `1/2/3/4` 参数读取与应用可执行
- [ ] `21` WARP 执行有过程输出
- [ ] `23/24/33/34` 有明确执行结果

### 7. 网络测试工具（重点）

- [ ] `4` DNS 解析有结果
- [ ] `21` TLS 证书链有 issuer/subject/dates
- [ ] `22` 端口连通有 open/closed 输出
- [ ] `23` 下载测速有速度和耗时
- [ ] `24` 网络汇总有接口/路由/连接统计
- [ ] `31` hyperspeed：显示“正在执行...”，并有完成/失败提示
- [ ] `32` nexttrace：显示“正在执行...”，并有完成/失败提示
- [ ] `33` IPv4/IPv6 有 OK/FAIL 输出
- [ ] `34` dig 延迟测试有输出或不可用提示

### 8. 安全与防护
